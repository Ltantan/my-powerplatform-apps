# このワークスペースのルール

## 基本方針

**このリポジトリは GitHub パブリック公開専用**。
Power Apps (.msapp) や Power Platform ソリューション (.zip) などの成果物を一般公開するための置き場。
開発・実験は行わない。`claude-lab`（プライベートリポジトリ）で行い、完成したものをここにコピーして公開する。

---

## このリポジトリでやること・やらないこと

| やること | やらないこと |
|---------|-------------|
| 完成した .msapp / .zip（ソリューション）/ スクリプトの配置 | 開発・実験 |
| README の整備・更新 | 個人情報・認証情報の管理 |
| リリースノートの追加 | プライベートデータの保存 |
| ライセンスの管理 | 未完成・動作未確認のものの公開 |

---

## 公開する成果物の種類

以下のいずれかを公開する。ファイル形式に合わせて README を書く。

| 種類 | ファイル形式 | 説明 |
|-----|------------|------|
| Canvas アプリ | `.msapp` | Power Apps でエクスポートしたアプリ単体 |
| ソリューション | `.zip` | 複数コンポーネントをまとめてエクスポートしたパッケージ |
| スクリプト等 | `.ps1` `.py` など | 自動化スクリプトなど |

---

## フォルダ構成

アプリ・ソリューション1件につき1フォルダ。

```
リポジトリルート/
├── CLAUDE.md             ← このファイル
├── README.md             ← リポジトリ全体の説明
└── アプリ名/
    ├── README.md         ← アプリの説明・使い方
    ├── *.msapp           ← Canvas アプリ（または *.zip ソリューション）
    └── assets/           ← スクリーンショット・説明画像など
```

## 個人情報・セキュリティのルール（最重要）

このリポジトリはパブリックであるため、以下を**絶対に守る**。

- **認証情報を入れない**: `credentials.json` `token.json` `.env` `*.key` は絶対にコミットしない
- **個人情報を入れない**: 氏名・メールアドレス・住所・電話番号・マイナンバーなどを含むファイルを置かない
- **テナント情報を入れない**: Microsoft 365 テナントID・クライアントID・接続文字列などを含むファイルを置かない
- **テスト用データを入れない**: 実在する人物・組織のデータをサンプルとして使わない
- **コミット前に必ず確認**: `git diff` で差分を確認してから push する

### Claude Code へのお願い
- コミット前に個人情報・認証情報が含まれていないか確認し、疑わしいものがあればユーザーに警告する
- `git push` 前に「パブリックリポジトリへの push です。個人情報が含まれていないか確認してください」と一言添える
- README 以外の Markdown ファイルを新規作成するときは、個人情報が含まれないよう注意する
- ユーザーは GitHub 初心者のため、git コマンドを実行するときは **何をしているか一言添えて**説明する（例:「リモートリポジトリに変更を送信します」など）
- 初めての操作（ブランチ・PR・リリース等）は手順を丁寧に説明する

## ファイルを追加・更新するときの手順

1. `claude-lab`（プライベート）で開発・テスト完了を確認する
2. 成果物をこのリポジトリの該当フォルダにコピーする（`claude-lab` からは削除しない）
3. README を最新の状態に更新する
4. `git diff` で差分を確認（個人情報・認証情報がないか）
5. コミット・push する

## README の書き方

アプリフォルダの `README.md` には以下を書く。

- **アプリの概要**（1〜2文）
- **スクリーンショット**（あれば）
- **インポート方法**
  - Canvas アプリ(.msapp): Power Apps → アプリのインポート → .msapp を選択
  - ソリューション(.zip): Power Platform 管理センター → ソリューションのインポート
- **必要な接続・権限**
- **使い方**（箇条書きでOK）
- **注意事項**（あれば）
